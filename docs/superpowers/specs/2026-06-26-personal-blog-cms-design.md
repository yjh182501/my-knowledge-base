# 个人博客后台改造设计

日期：2026-06-26

## 目标

把当前个人知识库从“运行目录写文件 + Git 自动同步”的模式，改造成适合阿里云 ECS 自部署的前后端分离个人博客系统。

成功标准：

- 在线后台可以直接写文章、保存草稿、发布文章。
- 文章数据不会因为网站重启、重新部署或 Git 同步失败而丢失。
- 前台只展示已发布且发布时间已到的文章。
- 后台支持文章编辑、删除、搜索、分类、标签和定时发布。
- 部署目标是用户自己的阿里云服务器，不再依赖 Render、Zeabur 等托管平台保存运行时数据。

## 当前问题

当前项目把文章保存到 `posts/` 目录，并在保存后尝试自动提交到 GitHub。这个模式有三个核心问题：

- 运行时文件不适合作为唯一数据源，部署平台重建后可能覆盖新增文章。
- Git 自动同步和平台自动部署容易互相影响，出现旧内容覆盖新内容。
- 当前代码把页面渲染、API、登录、文章读写都放在一个 `server.js` 中，后续加功能会越来越难维护。

## 推荐方案

采用“小型个人 CMS”方案：

- 前端：独立的后台和前台页面，负责展示、编辑和调用 API。
- 后端：独立 API 服务，负责登录、文章管理、搜索、上传和定时发布判断。
- 数据库：使用 SQLite，数据文件放在服务器固定数据目录。
- 上传文件：图片和附件保存到服务器固定上传目录。
- 部署：阿里云 ECS 上用 Nginx 暴露网站，用 PM2 或 Docker 运行后端。

这个方案比继续写 `posts/` 文件更稳定，也比直接上大型 CMS 更轻。

## 数据模型

第一版只需要这些核心表：

- `posts`
  - `id`
  - `slug`
  - `title`
  - `content`
  - `contentFormat`
  - `summary`
  - `coverImage`
  - `status`: `draft`、`published`、`scheduled`
  - `publishedAt`
  - `createdAt`
  - `updatedAt`
- `categories`
  - `id`
  - `name`
  - `slug`
- `tags`
  - `id`
  - `name`
  - `slug`
- `post_tags`
  - `postId`
  - `tagId`

暂不做多用户系统。个人自用只保留一个管理员密码。

## 功能范围

第一阶段包含：

- 管理登录
- 文章列表
- 新建文章
- 编辑文章
- 保存草稿
- 立即发布
- 定时发布
- 删除文章
- 分类和标签
- 前台文章列表
- 前台文章详情
- 全文搜索
- 图片上传
- 阿里云部署说明

第一阶段不包含：

- 评论系统
- 多用户权限
- 会员系统
- 复杂主题市场
- Obsidian 自动同步
- 微信公众号自动抓取

## 定时发布规则

后台保存文章时可以选择：

- 草稿：只在后台可见。
- 立即发布：前台立刻可见。
- 定时发布：设置 `publishedAt`，当前时间未到时前台不可见。

后端查询前台文章时只返回：

- `status = published`
- 或 `status = scheduled` 且 `publishedAt <= 当前时间`

这样不需要额外定时任务，也不会因为服务重启漏发布。

## 数据迁移

现有 `posts/` 中的 `.md` 和 `.html` 文件需要迁移进 SQLite：

- 文件名作为文章标题初始值。
- `.md` 文件保存为 Markdown。
- `.html` 文件保存为 HTML。
- 初始状态设为已发布。
- 迁移脚本只导入一次，避免重复生成文章。

迁移完成后，`posts/` 不再作为生产数据源，只作为历史备份保留。

## 部署方案

阿里云 ECS 上建议目录：

- 应用代码：`/opt/personal-blog/app`
- 数据库：`/opt/personal-blog/data/blog.sqlite`
- 上传文件：`/opt/personal-blog/uploads`
- 备份目录：`/opt/personal-blog/backups`

运行方式：

- Nginx 处理域名和 HTTPS。
- 后端监听本机端口。
- 前端静态文件由 Nginx 或后端提供。
- PM2 或 Docker 保持服务常驻。
- 每日备份 SQLite 数据库和上传目录。

## 验证标准

实现完成后需要验证：

- 重启服务后，文章仍然存在。
- 新增文章能保存为草稿。
- 立即发布后，前台可见。
- 定时发布的文章在时间到达前不可见，时间到达后可见。
- 编辑文章不会生成重复文章。
- 删除文章后前台不可见。
- 现有两篇文章能成功迁移。
- 阿里云部署文档能按步骤启动服务。
