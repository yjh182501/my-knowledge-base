一、AI 与智能体

围绕大语言模型（LLM）的核心概念，帮你理解 AI 的工作原理与应用方式。

概念 一句话理解
LLM（大型语言模型） 只有概率，没有灵魂，AI 的"发动机"
Context（上下文） AI 当前"看到"和"记住"的信息范围
Prompt（提示词） 你向 AI 发出的指令或问题
Memory（记忆） 让 AI 跨会话记住信息的能力
Agent（智能体） 自动干活的程序，AI 世界的"传话筒"
SubAgent（子智能体） 大 Agent 手下的小 Agent，分工协作
RAG（检索增强生成） 给 AI 装上"外接硬盘"——从向量数据库检索相关内容辅助生成
MCP（模型上下文协议） AI 和外部工具沟通的标准"接头暗号"
Web Search（联网搜索） AI 实时上网查资料
Workflow（工作流） 多个步骤串联起来的自动化流程
SKILL（技能） 封装好的特定能力模块，AI 可以按需调用
ComfyUI 开源节点式 AI 工作流工具，像"搭积木"一样编排图像/视频生成

二、编程语言与基础概念

前端三剑客

概念 比喻 作用
HTML 房子的毛坯结构 决定页面「有什么、怎么布局」
CSS 装修 + 软装 决定页面「好不好看」
JavaScript（JS） 水电 + 智能开关 决定页面「怎么交互、怎么运作」

CSS 预处理器

概念 比喻
原生 CSS 徒手盖房子，重复代码多
Sass / Less 专业施工队，有变量、循环等"工具"，编译后生成标准 CSS

通用概念

  • API： 软件世界里的"服务员/跑腿小哥"，在两个软件之间传递消息。就像插座——别人做好了功能，留一个口给你插上就能用。
  • SDK： 别人做好的半成品工具包，拿来直接用，不用从零造轮子（好比厂家给的"电机 + 驱动 + 说明书"套装）。
  • JSON： 程序世界的"通用清单/表格"，只存数据不跑代码（.json = 记事本，.js = 能跑的代码）。
  • 端口（Port）： 电脑里每个程序的"门牌号"——IP 地址是整栋大楼的地址，端口是每个房间的门牌号。
  • demo： 示例 / 小样。
  • init： 初始化。

三、前端开发框架与生态

Vue 生态（国内主流，中后台 / 企业项目首选）

概念 比喻 说明
Vue 3 前端界面"骨架搭建工具" 数据变界面自动变，组合式 API（setup）更灵活
Vue Router 手机导航 APP 管理页面跳转（如首页 → 个人中心）
Pinia 手机云盘 管理全局共享数据（如登录状态），替代 Vuex
Naive UI 手机主题商店 提供现成按钮/表格/弹窗组件，不用自己写样式
Vite 手机快充 开发热更新飞快，打包速度也快

核心搭配：Vue 3 + TypeScript + Vite + Vue Router + Pinia

React 生态（国外主流，大型复杂应用首选）

概念 比喻 说明
React 一套乐高积木 由 Facebook 维护，灵活性高
React Router 导航系统 管理页面路由
Zustand / Redux Toolkit 状态管理中心 管理全局数据

核心搭配：React 18 + TypeScript + Vite + React Router + Zustand/Redux Toolkit

进阶框架

  • Next.js： 基于 React 的"全栈 Web 框架"，带说明书、展示柜的"乐高主题展厅"——高性能、好 SEO。
  • Electron： 用网页技术（HTML/CSS/JS）开发桌面软件的框架，把网页的"美貌"和 Node.js 的"力量"装进 .exe。
  • WebAssembly（Wasm）： 给浏览器装的"涡轮增压发动机"——专门处理 JS 跑不快的重型计算任务。
  • React + TypeScript + Vite 组合：Vite 管构建/运行，React 管页面开发，TypeScript 管代码校验。

五、网络协议与通信

计算机之间"沟通的规矩"和"传输的管道"。

协议/概念 一句话理解 常见用途
HTTP 浏览器和服务器之间"通信的规矩" 网页访问
SSH 加密的远程命令行通道（端口 22） 远程管理服务器/NAS
端口 电脑里每个程序的"门牌号" 标识不同服务
DDNS（动态域名解析） 把变动的公网 IP 绑成固定域名，在外永远能访问家里设备 家庭服务器/NAS 外网访问
DHCP 路由器自动分配内网 IP 的服务 设备连 WiFi 自动拿到 IP
SMB 局域网文件共享协议 同一 WiFi 下直接打开 NAS 里的电影
WebDAV 通用网络文件访问协议 AList 转云盘为 WebDAV，VidHub 直接播放
FTP 老牌文件传输协议 上传文件到服务器
NFS Linux/NAS 高速共享协议 NAS 给 Linux 设备高速传 4K 视频
DLNA 局域网投屏播放协议 手机把视频投到电视播放
SMTP 简单邮件发送协议 专门用来发邮件
frpc（内网穿透） 把局域网服务映射到公网，在外也能访问家里设备 配合 Emby/AList 在外访问
curl 只会跑腿的"特工浏览器"——帮你去某个地址拿或送数据 测试网络、传输数据

易混协议对比

协议 区别一句话
DHCP vs DDNS DHCP = 局域网自动分内网 IP；DDNS = 外网把变动的公网 IP 绑成固定域名
SMB vs WebDAV SMB = 局域网文件共享；WebDAV = 跨网络文件访问
FTP vs NFS FTP = 老牌通用传输；NFS = Linux 下高速传输

概念 说明

命令行命令

环境 示例命令

概念 一句话理解

文件类型

类型 一句话理解

.exe 可以是安装程序，也可以直接运行的软件，格式比较自由

九、容器与部署

Docker 体系

概念 比喻 说明
Docker "即插即用小环境"的玩具工厂 专门做隔离的运行环境
容器（Container） 装好所有零件的成品玩具 隔离的、可直接运行的迷你系统环境
镜像（Image） 做玩具的"模具" 容器的模板，存了程序运行的所有基础内容
Docker Compose 玩具套装的"一键组装说明书" 一个配置文件启动/停止所有容器
镜像仓库 存放"玩具模具"的网上商店 不用自己做模具，直接下载现成的

沙盒 vs 容器

概念 区别
沙盒 = 笼子 只要关住不乱跑就行，里面有没有吃的不管
容器 = 带食物、水、玩具的笼子 关住不乱跑 + 里面能直接生活、运行软件

内网穿透

  • frpc（内网穿透客户端）： 把家里局域网的服务（Emby、AList、NAS）映射到公网，在外网也能访问。家里没有公网 IP → 在 NAS 上运行 frpc → 外面就能访问家里的服务。
